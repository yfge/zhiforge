# 知乎热点回答（搜索热点 → 匹配知识库 → 写文章 → 发布）

## 任务
搜索知乎热点问题，根据知识库现状判断能否回答，写文章并发布。

**重要约束：**
- 全程自动执行，禁止使用 AskUserQuestion 询问用户
- 禁止中途暂停等待确认
- 遇到问题自行决策或跳过，不要问用户

---

## 第一步：获取已回答问题（去重用）

1. 使用 Chrome MCP 打开创作中心文章列表：`https://www.zhihu.com/creator/manage/creation/article`
2. 获取已发布文章的标题列表，记录下来用于去重
3. 同时检查回答列表：`https://www.zhihu.com/creator/manage/creation/answer`
4. 合并成"已回答问题关键词列表"

---

## 第二步：搜索知乎热点问题

1. 读取 `CLAUDE.md` 了解作者专业领域和高匹配度话题
2. 使用 Chrome MCP 打开知乎搜索页面，搜索专业领域关键词：
   - `https://www.zhihu.com/search?type=content&q=AI编程`
   - `https://www.zhihu.com/search?type=content&q=Claude+Code`
   - `https://www.zhihu.com/search?type=content&q=vibe+coding`
3. 筛选高热度问题（关注数多、回答数少的优先）
4. 也可以查看热榜：`https://www.zhihu.com/hot`
5. **排除已回答过的问题**

---

## 第三步：评估知识库匹配度

1. 读取 `.claude/settings.json` 获取 `knowledge_base` 路径
2. 搜索知识库中的 .md 文件
3. 根据问题关键词匹配相关文章
4. **评估匹配度：**
   - ✅ **高匹配**：有直接相关博客，可以引用具体案例 → 继续
   - ⚠️ **中匹配**：有相关主题博客，需要扩展 → 继续
   - ❌ **低匹配**：没有相关内容，需要完全原创 → 跳过，找下一个问题

5. 输出匹配评估：
```
问题：[问题标题]
匹配度：高/中/低
相关博客：[博客文件名列表]
判断：可以回答 / 跳过
```

---

## 第四步：撰写文章

根据博客内容，撰写知乎文章：

**写作风格：**
- 开门见山，先给结论
- 用实际案例说话（引用博客中的项目经验）
- 技术内容要落地可执行
- 代码块使用 ```bash 或 ```python 标注语言
- 重点用 **加粗** 或 `代码格式`
- 结尾引导关注公众号或博客

**将文章写入临时文件：**
```
/private/tmp/claude-501/.../scratchpad/zhihu-article.md
```

---

## 第五步：发布文章并关联问题

### 5.1 打开专栏写作页面
**必须使用：** `https://zhuanlan.zhihu.com/write`

### 5.2 导入 Markdown 文件
1. 点击工具栏"导入"按钮
2. 点击"导入文档"
3. 使用 `upload_file` 上传 markdown 文件
4. 等待导入完成

### 5.3 填写标题
导入完成后，fill 标题框

### 5.4 设置投稿至问题
1. 点击"投稿至问题"下拉框
2. 在搜索框输入问题关键词（先 click 聚焦）
3. 点击"搜索"按钮
4. 点击目标问题的"选择"按钮
5. 点击"确定"或按 ESC 关闭对话框

### 5.5 发布
1. 点击"发布"按钮
2. 等待跳转（URL 包含 `just_published=1` 表示成功）

### 5.6 确认发布
访问创作中心确认：`https://www.zhihu.com/creator/manage/creation/article`

---

## 输出

完成后输出：
```
✅ 发布成功

问题：[问题标题]
热度：[关注数] 关注 / [回答数] 回答
文章：[文章标题]
链接：https://zhuanlan.zhihu.com/p/xxxxx
匹配博客：[博客文件名]
```

如果没有合适问题：
```
❌ 暂无适合回答的热点问题

已检查：
- [问题1] - 低匹配，跳过
- [问题2] - 已回答过，跳过
```

---

## 注意事项

- **禁止使用 AskUserQuestion**，全程自动决策
- **知识库匹配度低的问题要跳过**，不要强行回答
- 保持"老拐"风格：务实、不油腻、有干货
- 发布失败时重试一次，仍失败则输出错误信息
- 如果连续 3 个问题都不合适，结束并输出"暂无合适问题"
